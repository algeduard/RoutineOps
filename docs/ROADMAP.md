# Роадмап RoutineOps

Актуально на 2026-07-22 (v2.4.5).

Этот файл — единственный источник правды по тому, что уже есть и что планируется.
Матрица «Free vs Enterprise» в [README.md](../README.md) — её краткая выжимка.

## Принцип границы

- **Free** — весь операционный MDM для одного администратора или небольшой команды:
  инвентарь, группы, скрипты и расписания, софт-политики, блокировка экрана, карантин,
  временные админ-права, алерты, аудит, self-update агентов. Free должен быть **мощным**:
  это то, ради чего продукт ставят.
- **Enterprise** — то, что нужно **организации**: корпоративная идентичность, изоляция
  между подразделениями, деструктивные операции с ключами восстановления, compliance-
  отчётность, масштаб и HA.
- **Базовая безопасность не продаётся.** mTLS, аудит, шифрование секретов, подпись
  релизов — во Free. «Paywall security» мы не делаем.

Enterprise-код физически отсутствует в Free-сборке (сборочный тег), а не «выключен
флагом»: выключенную проверку в открытом коде просто удаляют форком.

## Планируется (Enterprise)

### Этап 1 — текущий

| Возможность | Комментарий |
|---|---|
| ✅ Лицензионное ядро | **Реализовано (в main).** Подписанный офлайн-файл лицензии (ed25519) + argon2id-пароль активации, feature-gate (`mgr.Has`), срок и grace; деплойер = центр лицензирования (свой ключ, CLI `routineops-license`). Сборка `-tags enterprise` |
| FileVault: эскроу ключей восстановления и принудительная блокировка (macOS) | Серверная часть готова, агентская в работе |
| ✅ SSO / OIDC | **Реализовано (в main).** Вход через внешний OIDC-IdP (authorization code + PKCE S256), за лицензией `FeatureSSO`. Верификация подписи/iss/aud/exp через go-oidc (JWKS с ротацией) + явная сверка nonce; server-side single-use state/nonce/verifier (`sso_auth_flows`); ленивый discovery (IdP down на старте не роняет сервер). Матчинг по неизменяемой (issuer, sub), НЕ по email — SSO не линкуется к существующему аккаунту (email-коллизия → отказ), `auth_source='oidc'` блокирует password-login и forgot/reset. JIT-провижининг (unusable-пароль) + role-mapping из claim (allowlist, fail-closed, it_admin по умолчанию никогда). Кнопка «Войти через SSO» на логине, dev-Keycloak в `deploy/`. go-oidc/oauth2 только под `//go:build enterprise`. SAML, back-channel logout, ручной релинк — follow-up |
| ✅ MFA / TOTP | **Реализовано (в main).** Двухшаговый вход: пароль → TOTP (RFC 6238, SHA1/6/30, совместимо с Google Authenticator/Authy/1Password). Секрет шифруется AES-256-GCM ключом `ROUTINEOPS_MFA_ENC_KEY` (env, не в БД, AAD=user_id); challenge между шагами — одноразовый DB-backed токен (не сессия). Recovery-коды (120 бит, sha256, одноразовые) + admin-reset залоченному. Replay-guard по last_step, per-account backoff на оба шага. **БАЗОВАЯ фича (не за лицензией)** — включается в профиле любым юзером. Потеря enc-ключа не блокирует вход (recovery/admin-reset). Enforce-by-policy (обязать MFA роли) — follow-up |
| SCIM-провижининг | Синхронизация пользователей из каталога |
| ✅ **Удаление ПО с устройства из интерфейса** | **Реализовано (в main).** Кнопка «удалить» в списке ПО карточки → тихая деинсталляция задачей агенту (MSI/QuietUninstallString/Inno) + подтверждение + аудит; за лицензией (`FeatureSoftwareRemoval`), гейтится в вебе через `/capabilities` |
| ✅ **Мультитенантность (MVP)** | **Реализовано (в main).** Модель тенантов (`tenants`, неудаляемый «Default» с фиксированным id) + привязка `tenant_id` на `devices`/`users` (nullable + DEFAULT=Default, `ON DELETE SET NULL` — существующие сущности бэкфилены, ничего не сломано). CRUD тенантов + назначение устройств/юзеров (`POST /tenants/{id}/assign`), счётчики, guard'ы (default неудаляем, непустой → 409, валидация slug), за лицензией (`FeatureMultitenancy`), страница «Тенанты». **Follow-up: ПОЛНАЯ per-query изоляция данных** (scoping каждого запроса к devices/users/tasks/… по tenant_id актора) — большой cross-cutting рефактор, здесь НЕ сделан |
| ✅ Экспорт аудита в SIEM | **Реализовано (в main).** Фоновый форвардинг событий аудита на webhook с HMAC-подписью, durable-курсор (safety-lag), за лицензией (`FeatureSIEMExport`), страница «SIEM-экспорт». syslog / CEF — follow-up |
| ✅ Tamper-evident аудит | **Реализовано (в main).** Кейд хеш-цепочка над `audit_log` (`entry_hmac = HMAC(key, prev‖canonical)`, ключ в env), подпись включается автоматически при заданном `ROUTINEOPS_AUDIT_HMAC_KEY` — ловит модификацию/удаление середины/вставку/replay (для подделки нужен ключ, которого нет в БД); голова цепочки (`audit_chain`) — якорь против наивного усечения хвоста. Проверка `GET /audit-log/verify` за лицензией (`FeatureAuditIntegrity`), кнопка на странице «Аудит». Полная защита от усечения хвоста — внешний якорь (SIEM-экспорт) |
| Compliance-дашборды и отчёты | CIS / SOC2-evidence, скоринг |
| Уровни критичности алертов и маршрутизация | Эскалация, дежурства |
| ✅ Сканирование ПО на уязвимости (CVE) | **Реализовано (в main).** Матчинг инвентаря `device_software` против фида уязвимостей: деплойер заливает выгрузку NVD/OSV (`POST /cve/feed` — JSON-массив, замена целиком; `DELETE /cve/feed`), `POST /cve/scan` пересобирает находки одним SQL-JOIN'ом (регистронезависимая подстрока имени + version-логика в Go: `<`/`<=`/`=`/`>`/`>=`, покомпонентное сравнение), `GET /cve/findings`/`/cve/summary`. Pending-устройства исключены, аудит `cve_feed_loaded`/`cve_scan_run`, за лицензией (`FeatureCVEScan`), страница «Уязвимости». Внешний сетевой фид / точный CPE-матчинг — follow-up |
| HA / несколько узлов, авто-бэкапы | Сейчас один узел = точка отказа |

## Как это устроено технически

- **Обновление сервера** — `git pull` + пересборка контейнеров; миграции накатывает
  отдельный сервис до старта сервера (fail-closed). См.
  [self-hosted-deploy.md](self-hosted-deploy.md).
- **Обновление агентов** — парк тянет подписанный бинарь с сервера **своего** деплойера.
  Мы не пушим ничего в чужую инфраструктуру.
- **Enterprise-доставка** — готовый образ сервера из приватного реестра по токену
  подписки; исходники Enterprise не отдаются. Агент у Enterprise-клиента — тот же
  публичный Free-агент, кроме FileVault-сценария.

Вопросы по Enterprise и предложения по роадмапу — через Issues репозитория.
