# Лицензионное ядро (Enterprise)

Актуально на 2026-07-22.

Механизм, включающий Enterprise-функции в сборке `-tags enterprise` и следящий за
сроком/лимитами. Open-core (`//go:build !enterprise`) физически не содержит этого кода —
там только стабы (`internal/license/doc_free.go`, `cmd/server/enterprise_stub.go`,
`cmd/routineops-license/main_free.go`), а роут `/license` отсутствует (404).

## Модель доверия: деплойер = центр лицензирования

RoutineOps ставится на **свои** серверы, часто офлайн, поэтому онлайн-проверок нет —
лицензия это **подписанный офлайн-файл**. Ключ подписи держит **деплойер** (владелец
форка), а не внешний вендор: тот же принцип «свой корень доверия», что у подписи релизов.

- **Приватный ed25519-ключ** — у деплойера, им `routineops-license` выпускает лицензии.
- **Публичный ключ** — вкомпилирован в enterprise-бинарь (`internal/license.defaultPubKey`,
  задаётся при сборке через `-ldflags "-X ...=<base64>"`), с override через
  `ROUTINEOPS_LICENSE_PUBKEY` (base64). Override — для деплойера, который держит СВОЙ
  корень доверия; в env его читает тот же оператор, что владеет сервером. Для раздачи
  бинаря НЕдоверенным третьим лицам ключ вшивают на сборке и env не пробрасывают
  (иначе оператор подставил бы свой ключ и выписал лицензию сам).

## Формат лицензии (blob)

Одна строка base64 (как в поле ввода веб-страницы). Внутри:

```
base64std( JSON{ "payload": "<base64std(canonical-json Claims)>",
                 "signature": "<base64std(ed25519 sig над байтами payload)>" } )
```

`Claims` (payload):

| Поле | Смысл |
|------|-------|
| `licensee` | кому выдана |
| `edition` | напр. `enterprise` |
| `features` | список фич; **пустой = вся редакция** (`Has` возвращает true на любой) |
| `seats` | устройств по договору (0 = без лимита; это метка, гейта по числу пока нет) |
| `license_id` | уникальный id (аудит, будущий revocation-list) |
| `issued_at` / `not_before` / `expires_at` | выпуск / начало / конец (zero = бессрочно) |
| `pw_salt` / `pw_hash` | argon2id соль+хеш пароля активации (пустые = без пароля) |

Подписываются **байты payload** (до base64). Проверка: `ed25519.Verify(pub, payload, sig)`.
Порядок полей фиксирован структурой Go → `json.Marshal` детерминирован (канонично).

## Пароль активации (argon2id)

Файл лицензии подписан (целостность/подлинность), но пароль активации — **второй фактор
для путей активации веб/env**: тот, у кого есть blob, но нет доступа к серверу, активировать
его не сможет (форма «Лицензия» и `ROUTINEOPS_LICENSE` требуют пароль). В `Claims` лежит
`argon2id(пароль, соль)` (соль + хеш, параметры фиксированы: time=3, memory=64 MiB,
threads=4, key=32 B). При активации сервер считает argon2id от введённого пароля и сравнивает
в постоянное время (`subtle.ConstantTimeCompare`). Пароль в открытом виде нигде не хранится и
не в blob'е. Брутфорс упирается в стоимость argon2id.

**Граница доверия персиста.** Уже активированный blob на rw-томе сервера (0600) на буте
поднимается БЕЗ повторного ввода пароля — это осознанный компромисс ради UX (не вводить
пароль каждый рестарт). Такой файл доверяется, потому что запись в license-том требует
доступа к самому серверу, а на этом уровне оператор уже владеет деплоем (модель self-hosted:
оператор = корень доверия). Пароль защищает сценарий «утёк файл, но не сервер», а не
«скомпрометирован хост». Подпись проверяется на КАЖДОМ пути (в т.ч. дисковом), поэтому
подсунуть НЕподписанный blob нельзя нигде.

## Срок и отсрочка (grace)

- До `not_before` → `configured && !valid`, UI: «Ещё не действует» (частая причина —
  отставшие часы VM).
- После `expires_at` → `!valid`, UI: «Истекла».
- `ROUTINEOPS_LICENSE_GRACE` (duration, напр. `720h`) продлевает `valid` на отсрочку
  после `expires_at`; UI показывает «на отсрочке — продлите».
- Часы — серверные (офлайн, NTP не гарантирован); это осознанный компромисс, ключ
  крутить назад бессмысленно (лицензию всё равно выписывает сам деплойер).

## Активация, персист, live-применение

- **Веб** (`POST /api/v1/license {license, activation_password}`, it_admin) — проверяет
  подпись+срок+пароль, применяет **сразу без рестарта**, сохраняет активированный blob на
  rw-том (`ROUTINEOPS_LICENSE_FILE`). Если том не пишется → 200 + `persist_warning`
  «применена, но не переживёт рестарт». Пустой `license` = деактивация (Free + удалить с
  диска). Отклонённый ключ НЕ сбрасывает текущую лицензию.
- **Boot** — грузит активированный blob с диска (уже проверенный, пароль повторно не
  нужен: доверяем своему диску) или `ROUTINEOPS_LICENSE`/`_FILE` из env (headless;
  `ROUTINEOPS_LICENSE_PASSWORD` для первой активации без веба).
- **Статус** (`GET /license`) — `LicenseStatus{configured, valid, licensee, edition,
  features, seats, expires_at, persist_warning}`.

## Feature-gate

`enterpriseSetup` кладёт активную лицензию в общий holder; Enterprise-функции спрашивают
`lic.Has("sso")` и т.п. Пустой `features` → true на любую фичу («вся редакция»). Пока
Enterprise-функций (SSO/SCIM/мультитенантность) нет, ядро задаёт механизм; они гейтятся
по мере появления. Смена лицензии в рантайме атомарна (holder под мьютексом/atomic).

## Границы редакций

Open-core собирается БЕЗ `-tags enterprise` и остаётся полноценным (весь операционный
MDM без лицензии). Enterprise-код в отдельных `//go:build enterprise`-файлах: `go build
./...` (free) их не видит, `go build -tags enterprise ./...` включает. `buf`/миграции не
затрагиваются — фича чисто серверная/крипто.
